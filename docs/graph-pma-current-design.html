graph-pma Current Design

2026-03-26 時点の新実装。old の VCSR / DGAP 系の考え方を取り入れつつ、 property subsystem と adjacency kernel を分離した現在の形をまとめたもの。

Adjacency-first 8-byte AdjacencyEntry VertexEntry index Physical locator + semantic edge id Property store / property index separated

Read Path

  • vertex-centric adjacency scan が主役
  • expand() は adjacency region を先に読む
  • exact-label expand は per-vertex label ranges を使う
  • EdgeId は semantic identity であり、primary traversal axis ではない

Write Path

  • graph mutation は graph-level op log に追記される
  • 一定閾値を超えると compact して base snapshot を再構築する
  • property store / property index は graph 本体とは別 subsystem

Identity Split

  • EdgeLocator: physical slot identity
  • EdgeId: stable semantic identity
  • locator_by_id: edge_id -> locator の橋

2. Stable Layout

Header version は 3。payload は node / edge / adjacency / property / property-index / op-log の順で並ぶ。

Field Meaning
node_bytes_len node catalog snapshot の長さ
edge_bytes_len edge payload snapshot の長さ
adjacency_bytes_len adjacency region の長さ
property_bytes_len property store snapshot の長さ
property_index_bytes_len property index snapshot の長さ
op_log_bytes_len graph-level mutation log の長さ

3. Adjacency Kernel

Physical Data

AdjacencyPma {
  outgoing_index: Vec<VertexEntry>
  outgoing_entries: Vec<AdjacencyEntry>
  outgoing_edge_ids: Vec<EdgeId>
  outgoing_label_ranges: Vec<VertexLabelRange>

  incoming_index: Vec<VertexEntry>
  incoming_entries: Vec<AdjacencyEntry>
  incoming_edge_ids: Vec<EdgeId>
  incoming_label_ranges: Vec<VertexLabelRange>

  locator_by_id: Vec<Option<EdgeLocator>>
  labels: AdjacencyLabelCatalog

  edges: Vec<Option<EdgeRecord>>
}

Core Types

Type Role
AdjacencyEntry(u64) 8-byte hot adjacency entry。下位 48 bit が target node id、上位 16 bit が packed meta。
VertexEntry ある頂点の adjacency slab と label-range 群の入口。old の VertexEntry に寄せた shape。
VertexLabelRange { label_id, start, len }。ある頂点内で 1 label に対応する subrange。
EdgeLocator { vertex, ordinal, direction }。adjacency position に基づく physical id。

4. VertexEntry Meaning

VertexEntry {
  node_id: NodeId,
  edge_index: u32,
  degree: u32,
  label_range_start: u32,
  label_range_len: u32,
  log_offset: i32,
}
  • edge_index: adjacency entry slab 内の開始位置
  • degree: その頂点の可視 edge 数
  • label_range_start, label_range_len: label-range table 内の範囲
  • log_offset: 将来の DGAP-style overflow / segment log の受け皿。現時点では -1

node_id を持っているのは現在の検索実装都合。dense vertex array へ寄せれば将来的には不要になりうる。

5. AdjacencyEntry Meaning

AdjacencyEntry(u64)

bits  0..47  target node id
bits 48..63  meta
  • target node id: 48-bit packed
  • meta: 16-bit packed label / flags
  • 現在の meta は主に label_id と tombstone 予約 bit

EdgeId 自体は adjacency entry には載せず、parallel sidecar outgoing_edge_ids / incoming_edge_ids に保持する。

6. Label-aware Expand

flowchart LR
    Q["expand(from, direction, label)"] --> V["VertexEntry lookup"]
    V --> L{"label specified?"}
    L -->|No| R1["use edge_index..edge_index+degree"]
    L -->|Yes| R2["scan VertexLabelRange slice\nfor matching label_id"]
    R2 --> R3["use label subrange only"]
    R1 --> E["AdjacencyEntry[] slice"]
    R3 --> E
    E --> X["EdgeId sidecar + EdgeLocator"]
    X --> P["EdgeRecord payload if needed"]
    E --> T["target node id from AdjacencyEntry"]
1. node_id で VertexEntry を引く
↓
2. label 指定がなければ edge_index..edge_index+degree を使う
↓
3. label 指定があれば、label_range_start..label_range_start+label_range_len の中から label_id が一致する VertexLabelRange を探す
↓
4. その subrange だけを adjacency slab から読む
↓
5. 必要なときだけ EdgeId から edge payload を引く

7. Identity Model

flowchart TD
    A["AdjacencyEntry\nphysical hot entry"] --> L["EdgeLocator\nphysical slot identity"]
    E["EdgeId\nsemantic identity"] --> L
    E --> P["Property store / property index / DML target"]
    L --> S["adjacency-local update / traversal position"]
Identity Meaning Used For
EdgeLocator physical slot identity adjacency-local update path, physical access
EdgeId semantic stable identity DML target, property store, public/kernel contract

現在は locator_by_id が edge_id -> locator を提供する。 これにより、read/write path は徐々に physical locator ベースへ寄せられる。

8. Property Subsystem

Property Store

  • PropertyStoreRuntime
  • backend: AppendLog / AbTree
  • node/edge properties を adjacency とは別に保持

Property Index

  • PropertyIndexRuntime
  • backend: AppendLog / AbTree
  • equality property lookup を支える secondary index

ABTree

  • byte-kv oriented tree
  • insert / get / delete / prefix scan / snapshot encode-decode
  • property subsystem の stable-memory backend 候補

9. Current Tradeoffs

  • VertexEntry.node_id は dense array 化すれば将来外せる
  • label_range_* は exact-label expand のために有効だが、別の vertex-label index へ分離する余地がある
  • log_offset はまだ live ではない。DGAP-style overflow log は未実装
  • adjacency snapshot は region として persisted されるが、open 時には memory 上へ fully reconstruct している

10. Next Natural Steps

  1. log_offset を実際に使う adjacency overflow / update log を導入する
  2. VertexEntry を dense vertex array 化して node_id を外すか検討する
  3. tombstone bit を live 化して delete path を adjacency-local にする
  4. adjacency region を page-oriented / lazy-load 化する
